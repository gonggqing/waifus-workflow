# Waifu File Structure Reference

## Folder Path Conventions

**Slug convention:** Chinese names → pinyin (e.g., `小铃` → `xiao-ling`), English names → lowercase hyphenated. When unsure of romanization, ask or approximate and note it.

**Path depends on whether a worldview exists:**

If the character belongs to an existing worldview, create the folder inside the worldview directory:

```
worldview/[worldview-slug]/[waifu-slug]/
├── basics.md              ← character core: sections 1–6 + image prompts
├── [scene-slug].md        ← one file per scene
└── ...
```

If the character has no worldview (standalone / legacy), create at project root:

```
[waifu-slug]/
├── basics.md
├── [scene-slug].md
└── ...
```

Standalone is the exception. Most characters belong to a worldview — use the worldview path.

**Scene file slugs:** derive from the scene name the same way — e.g., 场景 A · 三号窗口 → `san-hao-chuangkou.md`, 场景 B · 魔物逼近王都 → `mowu-bijin-wangdu.md`. If the name is short and clear in Chinese, a simple romanization is fine — ask the user if unsure.

## basics.md Structure

Contains character core data (narrative definition only — no image prompts). Image prompts live in the separate `concept-arts.md` file.

```
# [角色名]

---

## 1. 基本信息 / Basic Info

## 2. 世界观 / Worldview          （如有）

## 3. 角色简介 / Character Summary

## 4. 背景介绍 / Background

## 5. 开场白 / Opening Line

## 6. 初始对话 / Seed Dialogue

## 7. 场景 / Scenes            （各场景为独立文件，此处不写内容）
```

Section 2 (世界观 / Worldview) is omitted if the character has no world context. Section 7 (场景 / Scenes) lives in the individual scene files, not in basics.md.

---

## concept-arts.md Structure

A dedicated file for all image generation prompts related to this character. Produced by waifu-generator (Stage 2 of create-waifu pipeline). Contains character bases, portrait variants, and scene illustration prompts — grouped by platform.

```
# [角色名] — 视觉概念 / Concept Arts

## 角色基底 / Character Bases
[Base prompts — shared visual anchors prefixed into every portrait and scene prompt]

## 立绘组 / Portraits
[Portrait variants — front, three-quarter, casual, key emotion, etc.]

## 场景插图 / Scene Illustrations
[Per-scene prompt variants — wide vs close-up, calm vs peak tension, etc.]

## 备注 / Notes
[Inferred appearance details, recommended models/LoRA]
```

This file lives alongside basics.md in the character folder:

```
worldview/[worldview-slug]/[character-slug]/
├── basics.md              ← character narrative: sections 1–7
├── concept-arts.md        ← character + scene image prompts (from waifu-generator)
├── [scene-slug].md        ← one file per scene
└── ...
```

## Scene File Structure

Each scene gets its own file using the four-field format:

```markdown
# [场景名称]

**场景描述：** [one-line premise]

**开场白：**
"[opening line]"

**场景内容：**
[scene background — max 600 chars]
```

Leave `<!-- TODO -->` on incomplete fields rather than omitting them.

---

## Section Formatting Rules

### 1. 基本信息 / Basic Info

```markdown
### **1. 基本信息 / Basic Info**

- **名字:** [name]
- **标签:** [tag1, tag2, ...]
```

### 2. 世界观 / Worldview

Write as prose paragraphs or a short bullet list. No tables. Keep to ~500 chars.

**Source:** Copy the 世界摘要 / World Abstract from the worldview's `world.md` directly. Do not re-summarize or paraphrase — the abstract is already sized and tuned for character-card use.

### 3. 角色简介 / Character Summary

Single **简述:** line. One or two sentences — the elevator pitch.

### 4. 背景介绍 / Background

Four sub-sections, all Chinese labels:

```markdown
**角色定义**
[character definition paragraphs]

**外貌（不主动提及）**
- **[trait label]:** ...

**个性特质**
- **[trait label]:** ...
- [quirk bullets without sub-labels]

**互动风格**
- **(语气)**：...
- **(动作)**：...
- **(情绪)**：...
```

### 5. 开场白 / Opening Line

Single quoted line. No label needed beyond the section header.

### 6. 初始对话 / Seed Dialogue

Each exchange labeled **场景N：[topic]**, using `{{user}}:` / `**[角色名]:**` format.

### 7. 场景 / Scenes

Each scene uses the four-field structure:

```markdown
#### 场景 [X] · [name]

**场景描述：** [one-line premise]

**开场白：**
"[opening line]"

**场景内容：**
[scene background — max 600 chars]
```

Leave a `<!-- TODO -->` comment on incomplete scenes rather than omitting the block.

### Image Prompts → concept-arts.md

Character and scene image prompts are **not** written in basics.md. They go in a separate `concept-arts.md` file in the same character folder, generated by waifu-generator. See the concept-arts.md Structure section above for the format.

**Missing prompts:** use `<!-- TODO: run waifu-generator -->` as a placeholder in concept-arts.md rather than omitting the file.
