# World Building

A skill for creating and maintaining `./worldview/[slug]/` — living directories that grow richer with each new character and story.

---

## File Structure

Each worldview lives in its own folder with 4 files. The split keeps fast-changing data (characters, events) separate from the stable core (world rules, culture).

```
worldview/[slug]/
├── world.md        ← 世界摘要 (World Abstract) · 世界概览 (World Overview) · 世界规则 (World Rules) · 文化细节 (Cultural Details)
├── factions.md     ← 势力与派系 (Factions & Alliances) · 重要地点 (Key Locations)
├── timeline.md     ← 历史时间线 (Timeline)
├── characters.md   ← 角色档案 (Character Registry)
└── concept-art/    ← 世界视觉基底 (generated by world-concept-art skill)
```

**Slug convention:** world name in pinyin or English, hyphenated.
- 赛博朋克·璃光市 → `cyberpunk-liguang`
- 奇幻七王国大陆 → `fantasy-seven-kingdoms`
- 现代都市 → `modern-city`

## File Templates

### world.md

```markdown
# [世界名称]

> [一句话概括这个世界的气质和基调]

---

## 世界摘要 / World Abstract

[≤300 字的浓缩世界画像：一两句点明世界类型与基调，一两句概括核心矛盾或独特设定，一句提及主要势力格局。目标读者是对话引擎和角色卡——读完即可对齐世界调性。复杂世界可放宽至 500 字，但绝不超过。每次世界发生结构性变化时整篇重写，而非追加。]

---

## 世界概览 / World Overview

[完整描述：地理范围、时代背景、世界基调、重要背景知识。200–600 字起步，随角色和世界观丰富可持续扩展，无硬性上限。不是追加日志——当新信息改变了世界面貌时，应当整合重写，让读者始终拿到一篇连贯的描述。]

---

## 世界规则 / World Rules

[底层逻辑：魔法体系 / 科技水平 / 超自然法则 / 社会规则。
列点或短段，每条简明扼要。]

---

## 文化细节 / Cultural Details

[质感细节：习俗、禁忌、俚语、日常生活。碎片积累，不需系统。]
```

### factions.md

```markdown
# [世界名称] · 势力与地点 / Factions & Locations

## 势力与派系 / Factions & Alliances

### [派系名称]

## 重要地点 / Key Locations

### [地点名称]
```

```markdown
---
```

## Two Modes of Operation

### Mode 1 — 创建新世界观 / Create New World

Triggered when no worldview folder exists for this world.

**Input:** A character's `basics.md` (specifically the 世界观 / Worldview section)

**Process:**
1. Create `./worldview/[slug]/`
2. Build all 4 files from the character's 世界观 section:
   - `world.md` — write 世界摘要 / World Abstract (≤300 chars condensed snapshot) + 世界概览 / World Overview (full description) + 世界规则 / World Rules + 文化细节 / Cultural Details. The abstract is a distilled projection of the overview, not a copy-paste.
   - `factions.md` — extract any organizations/locations mentioned
   - `timeline.md` — extract any historical events from the backstory
   - `characters.md` — add this first character as the initial entry, with file path `worldview/[slug]/[character-slug]/basics.md`
3. The character folder itself (`worldview/[slug]/[character-slug]/`) is created by create-waifu's file assembly stage — world-building only creates the 4 world files alongside it.
4. Enrich where the source gives clear signals. For anything not established, leave a `<!-- 待扩展 -->` placeholder rather than inventing.

### Mode 2 — 扩展现有世界观 / Expand Existing World

Triggered when the worldview folder already exists and a new character has just been created.

**Input:** `./worldview/[slug]/` (all 4 files) + new character's `basics.md`

**Process — update only the files that changed:**

- **characters.md** — always: append the new character's entry
- **factions.md** — if the character belongs to a new faction or references a new location: append only the new entries; never rewrite existing ones
- **timeline.md** — if the character's backstory references a new historical event: append it in chronological order
- **world.md · 世界摘要 (World Abstract)** — rewrite the entire abstract on structural changes (new faction, new rule, tonal shift) so it always reflects the world's current state. ≤300 chars; complex worlds ≤500 chars
- **world.md · 世界概览 (World Overview)** — when a new character introduces world information not yet covered, **integrate-rewrite** the overview paragraph to stay coherent (weave into existing prose, don't append at the end). Leave untouched if no new info
- **world.md · 世界规则 (World Rules) / 文化细节 (Cultural Details)** — append only when the new character introduces genuinely new rules or cultural details

factions.md and timeline.md remain append-only. world.md's abstract and overview may be rewritten to stay coherent. Flag contradictions with `<!-- ⚠️ 待协调：[issue] -->`.

## Context Loading (for create-waifu)

When create-waifu needs worldview context before character generation, load files in this order:

1. **Quick context (always):** read the **世界摘要 / World Abstract** section from `world.md` + `characters.md` — enough for waifu-stories to align with the world's tone
2. **Full context (always):** read the full `world.md` (世界概览 / World Overview + 世界规则 / World Rules + 文化细节 / Cultural Details)
3. **Read if character has faction affiliation:** `factions.md`
4. **Read if character has historical backstory:** `timeline.md`

世界摘要 / World Abstract also serves as the source for each character's `basics.md` Section 2 (世界观 / Worldview) — copy it directly rather than re-summarizing.

## Expansion Principles

**Stay grounded.** Only write what the characters and stories have established.

**Grow incrementally.** A worldview that starts at 300 words and grows to 2000 over ten characters is better than one front-loaded with invented lore that later conflicts.

**Prefer specific over general.** "企业财团控制上层" is less useful than "七大企业财团中，赤核集团控制璃光市70%的义体供应链". Specifics come from backstories — pull them out.

**Flag contradictions.** When two characters' backstories don't align, mark the tension and let the user resolve it.

**Write as a native Chinese speaker.** World descriptions, location prose, and faction summaries should read with natural rhythm and texture — not like database entries or translated copy. Each world has its own atmosphere; let the prose reflect it:
- A cyberpunk world: clipped, neon-lit, cynical undertone
- A xianxia world: elevated register, unhurried, weight of centuries
- A modern city world: casual, observational, detail-specific
- A dark fantasy world: dense, myth-adjacent, portentous

If two worldviews read identically in style, at least one is wrong. Voice is part of world-building.

## World Concept Art Handoff

After Mode 1 (创建新世界观) finishes writing `world.md` and `factions.md`, or after Mode 2 (扩展现有世界观) adds new locations or significantly changes world descriptions, call the **world-concept-art** skill to generate or update environment/scene image prompts:

- **Mode 1 (new world):** After all 4 world files are created, call world-concept-art to generate the initial `concept-art/` folder with environment prompts derived from world.md and factions.md.
- **Mode 2 (expansion):** If factions.md gained new locations, or world.md's overview/rules changed in ways that affect visual appearance (new geography, new architectural style, new atmospheric phenomena), call world-concept-art in **append mode** to add prompts for the new locations without rewriting existing ones.
- **Mode 2 (no visual changes):** If the expansion only added characters, timeline events, or cultural details that don't affect environment visuals, skip the world-concept-art call.

This ensures the world's visual base stays in sync with its narrative description.
